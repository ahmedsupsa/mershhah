
# خريطة مشروع مائدتي (Mershhah)

## 1. وصف عام للمشروع

"مرشح" هو مركز نمو سحابي للمطاعم والمقاهي في السعودية. المشروع مبني باستخدام Next.js و Firebase، ويعتمد على بنية من ركيزتين: واجهة عميل ذكية وموحدة تجمع بيانات التفاعل، ومنصة أدوات داخلية لأصحاب المشاريع لتحليل هذه البيانات واتخاذ قرارات مدروسة.

**البنية الأساسية:**
1.  **الواجهة الذكية (Data Funnel):** صفحة عامة (`/hub/[username]`) لكل مشروع تعمل كبديل لـ "Linktree"، وتجمع كل الروابط (منيو، عروض، توصيل، فروع) في مكان واحد. هذه الواجهة هي المصدر الرئيسي لجمع بيانات تفاعل العملاء.
2.  **منصة الأدوات (Growth Engine):** لوحة تحكم خاصة بصاحب المشروع تحتوي على "صندوق أدوات" من الميزات التي تستخدم البيانات المجمعة لتقديم رؤى قابلة للتنفيذ (مثل تحليل المنيو، اقتراح عروض، تحليل التقييمات).

**الوظائف الأساسية:**
- **نظام مصادقة وتسجيل:** يعتمد على Firebase Auth لتسجيل أصحاب المطاعم والمقاهي.
- **لوحة تحكم للمدير:** إدارة المشتركين وحالة حساباتهم.
- **لوحة تحكم لصاحب المشروع:** الوصول إلى أدوات النمو، تخصيص الواجهة الذكية، وإدارة محتوى المشروع.
- **قدرات الذكاء الاصطناعي (AI):** استخدام Genkit و Gemini لتشغيل أدوات التحليل، إنشاء المحتوى، وتقديم رؤى ذكية.
- **الأمان:** يعتمد على Firestore Security Rules لضمان أن كل صاحب مشروع يمكنه الوصول إلى بياناته فقط.

## 2. شجرة المجلدات الأساسية وشرحها

```
.
├── src/
│   ├── app/                # الصفحات والمسارات (App Router)
│   ├── components/         # مكونات React القابلة لإعادة الاستخدام
│   ├── services/           # دوال للتواصل مع قاعدة البيانات (Firebase)
│   ├── ai/                 # منطق الذكاء الاصطناعي (Genkit Flows)
│   ├── lib/                # أدوات مساعدة، أنواع البيانات (types)، وعملاء Firebase
│   ├── hooks/              # React Hooks مخصصة (مثل useUser)
│   └── blog/               # محتوى وبيانات المدونة
├── docs/                   # ملفات التوثيق (مثل backend.json)
└── public/                 # الملفات الثابتة (غير موجود حالياً)
```

### شرح وظيفة كل مجلد:

- **`src/app`**: قلب التطبيق الذي يعتمد على Next.js App Router. يحتوي على مجلدات فرعية تنظم المسارات حسب الأدوار (`admin`, `owner`).
- **`src/components`**: يحتوي على جميع مكونات React، مقسمة إلى `ui` (مكونات ShadCN), `shared` (مشتركة), `auth`, `admin`, و `dashboard`.
- **`src/services`**: طبقة الوصول للبيانات. يحتوي على دوال `async` للتفاعل مع Firestore.
- **`src/ai`**: مركز عمليات الذكاء الاصطناعي، ويحتوي على "تدفقات" Genkit.
- **`src/lib`**: مكتبة الأدوات المساعدة، وأهمها `types.ts` (تعريفات TypeScript) و `firebase.ts` (إعدادات عميل Firebase).
- **`src/hooks`**: يحتوي على React Hooks مخصصة، وأهمها `useUser` لجلب بيانات المستخدم وملفه الشخصي.
- **`docs`**: يحتوي على ملفات توثيقية مثل `backend.json` الذي يصف مخطط قاعدة البيانات.

## 3. المكونات الرئيسية والمنطق التجاري

- **نقاط الدخول الرئيسية (Entry Points):**
    - `src/app/layout.tsx`: القالب الرئيسي للتطبيق.
    - `src/app/page.tsx`: الصفحة الرئيسية.
    - `src/middleware.ts`: يعترض الطلبات (حالياً لا يقوم بعمليات معقدة).

- **الصفحات الأساسية:**
    - **عام:** `/`, `/login`, `/register`, `/hub/[username]`, `/menu/[username]`.
    - **المدير:** `/admin/dashboard`, `/admin/management`.
    - **صاحب المشروع:** `/owner/dashboard`, `/owner/menu`, `/owner/customize`, `/owner/store`.

- **الخدمات (Services):**
    - **`src/services/restaurant.service.ts`**: يحتوي على دوال للتواصل مع Firestore من جهة الخادم.

- **المنطق التجاري (Business Logic):**
    - **المصادقة (Auth):** يتم التعامل معها عبر `Firebase Auth` ومكونات `src/components/auth`.
    - **التحقق من الصلاحيات:** يتم داخل `useUser` hook الذي يجلب بيانات المستخدم ودوره. تستخدم المكونات هذه البيانات لعرض المحتوى المناسب.
    - **إدارة البيانات:** تتم من خلال دوال Firebase SDK مباشرة في مكونات العميل (Client Components) لضمان الأمان والفعالية.

## 4. التقنيات المستخدمة

- **Framework:** Next.js (مع App Router)
- **لغة البرمجة:** TypeScript
- **قاعدة البيانات والمصادقة:** Firebase (Firestore, Auth, Storage)
- **الذكاء الاصطناعي:** Genkit, Google Gemini
- **الواجهة الأمامية (UI):** React, Shadcn/UI
- **التصميم والـ Styling:** Tailwind CSS
- **الأيقونات:** Lucide React
- **التحقق من النماذج:** React Hook Form, Zod

## 5. خريطة تدفق البيانات المبسطة (مثال: صاحب مشروع يعدل هوية مشروعه)

1.  **المستخدم (صاحب المشروع):** يزور صفحة `/owner/customize`.
2.  **React (Client Component):**
    -  يتم استدعاء `useUser` hook لجلب بيانات المستخدم والمشروع المرتبط به.
3.  **UI Rendering:** يتم عرض بيانات المشروع (الألوان، الشعار) في نموذج التعديل.
4.  **المستخدم:** يغير اللون الأساسي ويضغط "حفظ".
5.  **Client Logic:**
    - يتم تفعيل دالة `handleSaveSettings`.
    - الدالة تستخدم `updateDoc` من Firebase SDK لتحديث المستند في `restaurants` الذي يطابق `restaurantId`.
6.  **Firestore Security Rules:** سياسة الأمان في قاعدة البيانات تتأكد من أن المستخدم الذي يحاول التعديل هو بالفعل مالك هذا المشروع (`owner_id === request.auth.uid`) قبل السماح بالعملية.
7.  **Database:** يتم حفظ البيانات الجديدة في `Firestore`.
8.  **UI Update:** يتم عرض رسالة نجاح، ويتم تحديث واجهة المستخدم لتعكس التغيير الجديد.
